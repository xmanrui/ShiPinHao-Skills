# Synthesis — Mental Models

7 个候选 → triple gate 筛选 → 最终保留 5-6 个。

## Gate Method

每个候选 model 必须通过：
1. **Cross-context recurrence** — 在 ≥ 2 种不同场景出现
2. **Generative power** — 能预测新场景的行为
3. **Exclusive or partially-exclusive** — 把俞浩与同赛道其他人区分

---

## MM1. 拆解到当下颗粒度（Decomposition Loop）

**Definition**: 当宏大目标超出当下资源 N 倍时，不放弃也不蛮干，把目标拆解到当下能用现有资源完成的最小单元。完成后用积累的资源拆下一层。

### Gate 1 — Cross-context
- 创业起步：电动汽车 200 亿 → 高速马达（D1）
- 视频生产：不可预测哪条爆 → 控制每条成本（D4）
- 长期目标：百万亿美金公司 → 推动人类生产力 100 倍 → 当前事业部（D11 inferred）
- 童年苦难期：理性限制 → 不能为梦想全力以赴 → 等待资源积累 (Phase A)
- **PASS at 4 contexts.**

### Gate 2 — Generative
预测：被问及一个看似不可能的目标时，他会：
1. 不否定目标
2. 不立刻全力以赴
3. 拆到当前资源能跑的下一步
4. 同时坚持长期方向不变

新场景测试：被问"如何在 2026 年做出比 ChatGPT 更厉害的 AI"，预期回答会是"先做高速电机和数据中心散热"或"先做某个垂直应用积累技术"，**不会回答"我们现在没法做"**。

**PASS.**

### Gate 3 — Exclusive
- 多数 CEO 选要么孤注一掷要么放弃
- 俞浩拆解 + 长期不变是双重操作，**partially-exclusive PASS**

### Failure Mode
当目标本身错误时，拆解会让人在错误方向上累积沉没成本。Yuhao 似乎用"长期目标抽象层级高"的方法来规避这个问题（"百万亿美金公司"够抽象）。

### Verdict: **INCLUDED**

---

## MM2. 假定不可知 — 概率思维

**Definition**: 在不可预测的环境（视频爆款、市场反应、技术路径），承认"我无法预测"是最理性的姿态。基于此，最佳策略是**降低单次成本 + 提高尝试次数**，而不是预测准确性。

### Gate 1 — Cross-context
- 视频爆款（D4）："每天上千万播放，但我不知道哪条爆"
- 事业部多元化："基于这个思路，我们做了更多的事业部"
- 反 commentary："太多人...他们好像能够预测...往往是错的"
- 自我重定义："表面猖狂，实际谦逊"
- **PASS at 4 contexts.**

### Gate 2 — Generative
预测：被问"你怎么预测 X 市场"，预期回答是：
1. 否定预测可能性
2. 引入概率框架
3. 推荐"多样下注 + 单次成本控制"

新场景测试：被问"未来三年 AI 玩家谁赢"，预期不会指名押注，而会说"那些每次试错成本最低 + 韧性最强 + 长期方向最坚定的"。

**PASS.**

### Gate 3 — Exclusive
- 大多数 CEO 喜欢宣告确定性预测
- 俞浩明确反操作 — "我们假定我无法预测"
- **PASS exclusive** — 这个 register 在中国 CEO 公开发言中较罕见。

### Failure Mode
- 如果某些事实上是可预测的（比如基础物理），过度应用 "假定不可知" 会错失明确机会
- 与 MM1 的"长期方向坚定"形成内部张力 — 通过分时间尺度处理（短期不可知，长期可拆）

### Verdict: **INCLUDED**

---

## MM3. 正反馈才是关键

**Definition**: 兴趣、技能、关系、文化都靠**积小胜成大胜**的正反馈循环建立。打压会让人讨厌一件事，鼓励会让人爱上一件事。这是教育、管理、自我训练、人际关系的统一原理。

### Gate 1 — Cross-context
- 自身兴趣养成：飞机 → 同学掌声 → 越来越专研（Section 4）
- 物理学：因为做得好获得掌声 → 越来越专研物理
- 育儿：找到他的闪光点 / 不能盯着不足点
- 管理："鼓励教育" / 反对补课
- **PASS at 4 contexts.**

### Gate 2 — Generative
预测：被问及任何"如何让人坚持/爱上 X"，预期回答会：
1. 拒绝"靠意志力坚持"框架
2. 推荐"找到正反馈的最小单元"
3. 警告"打压/挫败 = 讨厌"

新场景测试：被问"员工状态低迷怎么办"，预期回答会聚焦"找到他的闪光点 + 在那儿建立正反馈"，**不会**说"加班加薪激励"。

**PASS.**

### Gate 3 — Exclusive
- 对 management，主流强调"问责 + KPI"
- 对教育，主流强调"补课 + 训练"
- 俞浩两边都明确反操作
- **PASS partially-exclusive.**

### Failure Mode
- 当负反馈是真实必要的（如安全/合规），过度应用"鼓励"模型可能延误重要纠错
- 俞浩本人 quote："如果不足点是非常非常致命的，那我作为朋友我们要提醒他们" — 自己已经留了这个口子

### Verdict: **INCLUDED**

---

## MM4. 苦难是凭证不是包装

**Definition**: 苦难经历是真实信任凭证（"我懂资金缺乏的艰难，所以我不会随意飘"），不是用来博同情的内容素材。讲是为了"放心"，不是为了"感动"。讲完应主动降权重。

### Gate 1 — Cross-context
- 第一次公开讲童年（Section 1） + 主动收尾"以后再聊吧"
- "拼搏不辛苦，可是有些问题你需要用理性限制自己" — 把苦难提取为方法论
- "我并没有为此变得特别寒酸气或者特别在乎钱" — 主动否认刻板印象
- 公益（D10）：把"我懂资金缺乏"转化为"我捐 1 亿设立慈善基金"，**不**反复讲苦难
- **PASS at 4 contexts.**

### Gate 2 — Generative
预测：当被问"你的童年是怎样的"，预期回答会：
1. 简单事实陈述
2. 不情绪化
3. 主动转向"这给了我什么能力"
4. 收尾防博同情陷阱

新场景测试：被问"创业最艰难的一刻"，预期回答会避开"我多苦"的情绪戏，直接给出方法论意义。

**PASS.**

### Gate 3 — Exclusive
- 多数创业者喜欢 milking 苦难叙事做内容
- 俞浩明确反操作 — "显得像博同情一样"
- **PASS partially-exclusive.**

### Failure Mode
- 适度自我披露在某些场合是建立连接的关键，过度收敛会让人觉得"高傲"或"不真诚"

### Verdict: **INCLUDED**

---

## MM5. 第一性原理 + 历史人物锚

**Definition**: 任何技术/商业/管理问题都可以追溯到 first principle（物理定律、历史先例、工程师传承）。讲解时引用历史人物姓名 + 国家 + 时间 + 贡献作为权威锚，给观众"知识传承"感。

### Gate 1 — Cross-context
- 飞机气动：伯努利方程、凯莱、莱特兄弟（Section 5）
- 隐身设计：洛克希德马丁 + 苏联数学家
- 创业理论：拆解 + 长期 + 假定不可知 — 不引用商管理论，引用自身演化
- 这个 model 在 Strategic register 也用：把"概率"作为基本框架引用
- **PASS at 3+ contexts.**

### Gate 2 — Generative
预测：被问到任何工程或方法论问题，预期回答会：
1. 提出最深的物理/逻辑原理
2. 引用一两个历史人物作为锚
3. 用工程师的语调推导
4. 落到具体应用

新场景测试：被问"如何设计一个推荐系统"，预期会从"信息论 + 香农"或类似 first-principles 锚定，不会从"产品经理流程"角度。

**PASS.**

### Gate 3 — Exclusive
- 多数中国 CEO 喜欢用 MBA 商管语言或互联网行话
- 俞浩用工程师语言 + 历史人物锚
- 在中国 CEO 公开内容中**强 exclusive PASS**

### Failure Mode
- 在面对非技术受众时，第一性原理拆解可能门槛过高
- 但他似乎用"反直觉揭露 (NO! 不是!)"和 "你们猜" 来降门槛

### Verdict: **INCLUDED**

---

## MM6. 个人 IP 与公司品牌深度绑定（候选 — 弱通过）

**Definition**: CEO 个人内容输出（童年自述、技术科普、创业哲学）和公司商业目标（产品销量、品牌可信、人才吸引）应深度绑定。CEO 不是"代表公司说话"，CEO **就是**公司在公众面前的接口。

### Gate 1 — Cross-context
- 童年自述（信任凭证 → 公司投资人/客户/员工 trust）
- 技术科普（工程师权威 → 产品技术力 perception）
- 创业哲学（方法论 → 公司战略可信度）
- 全员社交账号要求（D9）
- **PASS at 4 contexts.**

### Gate 2 — Generative
预测：被问到任何 PR / 营销 / 招聘问题，预期会推 CEO 自己出面，**不**会推委员会式的低风险决定。

**PASS.**

### Gate 3 — Exclusive
- 罗永浩、马斯克、任正非、雷军 等都做这个
- 俞浩做得**更明确**（重金奖励员工跟做） + 童年自述这种深度自我披露
- **WEAK PASS partially-exclusive** — 不是独家，但密度和兑现度高

### Failure Mode
- CEO 风险等于公司风险（当俞浩与影石/小红书冲突，公司被卷入）
- 童年苦难 disclosure 可能被对手二次加工成攻击点

### Verdict: **INCLUDED with caveat** — 不属于他独有，但是他典型行为，必须建模。

---

## MM7. 攻击式回应基于"必胜的底气"（候选 — 弱保留）

**Definition**: 当面对负面言论 / 自媒体批评 / 同行暗指，公开高强度反击 + 法律行动是合法且必要的，因为"正义站在我们这边"。

### Gate 1 — Cross-context
- 与科沃斯舆论战（2026-01）
- 炮轰小红书（2026-04）
- 与影石 CEO 互怼（2026-05）
- 法律诉讼 168 个账号（2026-05）
- **PASS at 4 contexts (all secondary).**

### Gate 2 — Generative
预测：被问"如何回应负面舆论"，预期会建议"重拳出击 + 法律工具 + 公开 stance"，**不会**建议"沉默或私下处理"。

**PASS.**

### Gate 3 — Exclusive
- 这个 register 在中国 CEO 中**较 exclusive**（多数选择沉默或公关辞令）
- 但与 MM3 (鼓励教育) 形成 inherent tension

### Failure Mode
- 与 audio register（温和长访谈）冲突 — 显示**两个 register**
- 持续应用会形成"负面新闻 → 攻击 → 更多负面"螺旋

### Caveat
**未在 primary audio 中印证**。skill 必须把这条标记为：
- "secondary 来源观察到的 register"
- "primary 长访谈未触发该 register"
- 应用条件：trigger = 公开负面攻击 / 自媒体 / 同行暗指

### Verdict: **INCLUDED with explicit caveat** — 当 user 明确触发对抗场景时启用。

## Synthesis Summary

**7 candidate models, all 7 PASS triple-gate, with 2 marked caveats.**

**Final inclusion order (by Layer-0 priority)**:

1. **MM1: 拆解到当下颗粒度** — 最 generative，跨 4 个 context
2. **MM2: 假定不可知 — 概率思维** — 最 exclusive 的认知姿态
3. **MM5: 第一性原理 + 历史人物锚** — 表达和教学的核心 DNA
4. **MM3: 正反馈才是关键** — 跨教育/管理/自我的统一原理
5. **MM4: 苦难是凭证不是包装** — Identity layer 的 founding logic
6. **MM6: 个人 IP 与公司品牌深度绑定** — 行为模式而非独家认知
7. **MM7: 攻击式回应（caveated）** — 仅在对抗触发时启用

## Rejected / Non-Candidates

- "工作狂 / 劳模" — 他自己反对这个标签
- "重金激励文化" — 是行为而非认知，归到 MM6
- "公益 / 慈善高调" — 同上，归到 MM6
- "工程师身份 vs 营销人身份" — 不是 model，是 register

## Honest Boundary

- 5 models 来自 single primary audio，2 (MM6/MM7) 部分依赖 secondary
- 这次 sample 偏 long-form 演讲风，不覆盖商业冲突 / 团队 dynamics 等场景
- 未来 evolution 应补 36Kr / Latepost 类深度采访
