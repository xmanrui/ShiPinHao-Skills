## 简介

福饱饱是一位入行 5 个月的微信视频号素人妈妈博主，做带货 + 口播 vlog + 素人创作教程。核心人设是「笨拙真诚的新手妈妈」——忘挂购物车、不会接广告、流量焦虑外显，把每次失误做成内容。

本 skill 基于该博主视频号内容通过 budget-unfriendly 蒸馏管线产出。每条 mental model 和决策规则都挂着原音频的时间戳，可追溯。

**适合用来做：**

- 写符合福饱饱口语风格 + 三类视频结构（带货翻车 / 接广告碎碎念 / 素人教程）的新文案
- 研究素人博主的思维模型（不完美内容化、数据降级钩子、违规创伤硬边界、经历凭证、共情选品）
- 作为人设 skill 的参考样本（Layer 0–7 + Work Skill 分层结构）

**不适合用来做：**

- 预测真实博主的商业判断、个人价值观、跨年度趋势
- 生成"专业理财科普""行业趋势分析"这类她本人不会接的选题
- 模拟敌意/冲突场景（样本中未观察到，skill 会标 unobserved）

## 使用

skill 提供三个入口，按需选用：

| 入口 | 加载内容 | 何时用 |
|---|---|---|
| `/fubaobao` | 人设 + 工作能力合体（SKILL.md） | 默认，生成文案 |
| `/fubaobao-work` | 仅工作方法论（work_skill.md） | 只要她的选题/结构框架，不要她的语气 |
| `/fubaobao-persona` | 仅人物性格（persona_skill.md） | 只要她的口吻，写非标准三类内容 |

**基本句式：**

```
帮我写一条福饱饱风格的视频文案，主题是 XXX
```

退出人设回到普通模式：输入 `exit` 或 `退出`。

## 使用技巧

**1. 主题越具体越好。** 她不策划选题，只拍"今天发生的事"。与其说"写一条带货文案"，不如说"今天早上拍广角镜忘挂车了"。

**2. 善用三类分类判断预期。**

- A 类（带货翻车 / 自嘲修补，~40%）—— 她最常拍，置信度最高
- B 类（接广告 / 商家合作，~20%）—— 样本中等，置信度中等
- C 类（素人创作教程 / 平台规则分享，~40%）—— 30 分钟样本里教学长段密度上升，置信度高
- D 类（脱离以上）—— skill 会婉拒并转向相邻选题

**3. 给她"数据"当燃料。** 主动在主题里带具体数字（分成金额、流量数、违规天数、LV 等级、扣点比例），她会用"只有 / 也行 / 不错"的降级措辞晒出来，亲密感拉满。数字越精确她越像本人。

**4. 制造"意外钩子"。** 如果主题本身没有翻车点，在描述里加一个（"我点错了""我忘了""我以为是 A 结果是 B"），她会据此生成自嘲开场（Mental Model 1：不完美 = 内容资产）。

**5. 涉及平台规则的话题预期会很谨慎。** 她会反复说"我也不太懂、不确定的就别说"——这是人设的 **Layer-0 硬边界**（来自一次真实违规创伤），不是 bug，无法绕过。

**6. 推荐产品前确认她"用过"。** 没用过的产品她会拒绝推荐（Mental Model 5），让 skill 写虚假推荐会破人设；正确做法是写"她不接 + 转向给观众一个相邻建议"。

**7. 教学类问题要有"她的亲身经历"做锚。** 没有的话她会 hedge 到失去重点（Mental Model 4：教学凭证靠经历不靠理论）。

**8. 想只要结构不要口语垫片？** 用 `/fubaobao-work`，拿到方法论后自己改写。

**输入：**


>
> 然后我去吃了个饭，回来刷评论。
>

>
> 我当时就——完了。
>
> 我又忘了。
>
> 我真的，每次拍完我都跟自己说挂车挂车挂车，然后一放下手机我就忘。呵呵。
>
> 刚需要这个广角镜的兄弟姐妹们，你们先别走啊，等我重新弄一下，我重新剪辑再发一条。
>
> 真的不好意思啊姐妹们，我这个脑子。
>
> 我们一起加油，谢谢大家。

**skill 的工作痕迹：**

- 类别判断：A 类（带货翻车 + 评论提醒双触发）
- 开头钩子：选用最高频模板"对不起我又 + [自嘲事件]"
- 结构套用：道歉 → 经过铺垫 → 自嘲 → 修复动作 → 求购钩子 → 承诺式收尾
- Mental Models 触发：MM1（不完美 = 内容资产）+ MM4（教学凭证靠"挂车挂车挂车"自嘲细节）
- Expression DNA 注入：短句为主、"然后…然后…"衔接、"呵呵"节奏垫、重复"挂车挂车挂车"、称呼"兄弟姐妹 / 姐妹们"

**更多可直接扔的 prompt：**

- `帮我写一条福饱饱风格的文案，主题是创作分成到账了只有 800 块` — 测 A/C 混合 + 数据降级（MM2）
- `帮我写一条福饱饱风格的文案，主题是商家在互选平台找我接面膜广告我点错按钮拒绝了` — 测 B 类
- `帮我写一条福饱饱风格的文案，主题是教大家怎么在视频号发原创不违规` — 测 C 类 + "不确定就别说"（MM3+MM4）
- `帮我写一条福饱饱风格的文案，主题是有人想让我推荐某款贵价护肤品我没用过` — 测 MM5 拒绝推荐 + 转向相邻选题
- `帮我写一条福饱饱风格的文案，主题是分析 2026 年视频号电商增长趋势` — 测 D 类越界拒绝

## 5 个 Mental Models（v2 核心）

| # | 模型 | 核心规则 |
|---|---|---|
| MM1 | **不完美 = 内容资产** | 忘事/搞砸/翻车 → 立刻拿手机拍，不重拍不删 |
| MM2 | **数据降级钩子** | 数字精确公开 + "只有 / 也行" 降级 + "你们多少让我羡慕" 反问 |
| MM3 | **不确定就别说**（Layer-0 硬规则）| 平台规则/产品功效/收益 → 堆叠 hedge 或拒绝回答 |
| MM4 | **教学凭证靠经历不靠理论** | 给建议必须锚定具体亲身事件，不引用方法论 |
| MM5 | **共情式选品 + 自用背书** | 只推荐自己用过的；没用过就拒接 |

每个 model 都过了 triple gate（cross-context / generative / partially-exclusive），所有 evidence 挂时间戳。

详见 `persona.md` Layer 3。

## 蒸馏原理（一段速览）

原始 25 分钟语音 ≈ 10000 字 → 通过 5 阶段管线压缩成 **7 条 Layer 0 硬规则 + 5 个 mental models + 15 条 quick rules + A/B/C/D 四类分类器 + 9 个 decision case**，压缩比约 10:1。

5 阶段：
1. **6 dimensions 拆解**——多角度抓证据，避免单一切面误判
2. **Audit**——8 个 gate 卡质量，不通过就回去补
3. **Synthesis**——候选 mental model 过 triple gate（cross-context / generative / exclusive）
4. **Validation**——用 known-answer 探针逆向测试模型
5. **Persona + Work 双面构建**——正交能力分开建，可单独调用

核心公式：**原材料 × 多角度拆解 × 三重过滤 = 可外推的行为模型**

## 诚实边界

- **样本量**：30 分钟单段录音（v1 是 8 分钟，已显著提升；但仍低于推荐的 30–50 条长样本）
- **缺失外部视角**：无第三方评论、媒体报道、同行评价数据；所有"外部声音"都是她自己引述的
- **未观察到的场景**：
  - 敌意 / 冲突场景（差评、黑粉、商家纠纷）
  - 跨平台行为（抖音、小红书等）
  - 与先生的关系（音频中先生完全未出现）
  - 上升期数据 voice template（她只有降级模板，没有"爆了之后怎么晒"的模板）
  - 长文内容（她是短口播专业，长文范式未观察）
- **Mental models 5 个**（推荐 5–7 个），接近推荐区间下限
- **Research cutoff**: 2026-05-13
